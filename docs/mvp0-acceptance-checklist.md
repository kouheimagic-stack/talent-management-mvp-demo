# MVP 0 受け入れテストチェックリスト

対象: 一般社員向けプロフィール基盤

## ログイン

- [ ] ログイン画面で `misaki.sato@example.com / password123` を入力してログインできる
- [ ] ログイン後、一般社員は `/me` に遷移する
- [ ] 間違ったメールアドレスまたはパスワードではエラーが表示される
- [ ] ログアウトできる
- [ ] ログアウト後に `/me` へ直接アクセスするとログイン画面へ戻る

## 権限・ナビゲーション

- [ ] 一般社員のメニューは「マイページ」「プロフィール編集」「社員公開プロフィール一覧」「ログアウト」だけ
- [ ] 一般社員のメニューに「部下」「面談」「アクション」「人事管理」「CSV」「組織管理」が表示されない
- [ ] employee が `/manager` を開くと `/me` に戻る
- [ ] employee が `/hr` を開くと `/me` に戻る
- [ ] employee が `/admin` を開くと `/me` に戻る
- [ ] employee が `/actions` や `/meetings` を開くと `/me` に戻る

## マイページ

- [ ] 顔写真、氏名、所属、役職が表示される
- [ ] プロフィール完成度が表示される
- [ ] 公開中の項目数が表示される
- [ ] 非公開の項目数が表示される
- [ ] 「プロフィール編集」ボタンで編集画面に移動できる
- [ ] 「公開プロフィールプレビュー」ボタンで公開プロフィール詳細に移動できる

## プロフィール編集

- [ ] この画面で何をするか説明がある
- [ ] 自己紹介を入力できる
- [ ] 現在の仕事内容を入力できる
- [ ] 社内経歴を入力できる
- [ ] 保有資格を入力できる
- [ ] 得意領域を入力できる
- [ ] 伸ばしたいスキルを入力できる
- [ ] 希望キャリア公開用コメントを入力できる
- [ ] 希望キャリア非公開コメントを入力できる
- [ ] 異動希望を入力できる
- [ ] 面談前メモを入力できる
- [ ] 顔写真をアップロードするとプレビューされる
- [ ] 顔写真を削除できる
- [ ] 入力内容を変更すると「未保存の変更があります」と表示される
- [ ] 保存中は「保存中...」が表示される
- [ ] 保存後は「保存しました」が表示される
- [ ] 保存後にリロードしても内容が残る
- [ ] 未保存変更がある状態でページ移動すると警告が出る

## 公開設定

- [ ] 公開中の項目は青または緑で目立つ
- [ ] 非公開項目はグレーで表示される
- [ ] 非公開固定項目には「非公開固定」と表示される
- [ ] 非公開固定項目に公開トグルが表示されない
- [ ] 非公開から公開に切り替える時、確認モーダルが表示される
- [ ] 確認モーダルに「この項目を公開すると、全社員が閲覧できるようになります。公開してもよろしいですか？」と表示される
- [ ] 公開から非公開に戻す時は確認なしで戻る

## 公開プロフィールプレビュー・詳細

- [ ] プレビューに「これは他社員から見える内容です」と表示される
- [ ] 編集中の内容がプレビューに反映される
- [ ] 非公開項目はプレビューに表示されない
- [ ] 空欄の公開項目はプレビューに表示されない
- [ ] 非公開固定項目はプレビューに表示されない
- [ ] 公開プロフィール詳細に非公開情報が表示されない
- [ ] 公開プロフィール詳細から自分の編集画面へ戻れる

## 社員公開プロフィール一覧

- [ ] 「社内の人を探す」目的が分かる説明がある
- [ ] 氏名、得意領域、資格、自己紹介で検索できる
- [ ] 部署で絞り込める
- [ ] 資格/得意領域で絞り込める
- [ ] 公開情報が少ない社員は空欄が自然に隠れる
- [ ] 顔写真が非公開または未設定の場合、デフォルトアイコンが表示される
- [ ] 「公開プロフィールを見る」ボタンで詳細へ移動できる
- [ ] 異動希望、面談前メモ、希望キャリア非公開コメント、評価、給与、面談記録、AI分析が表示されない

## スマホ表示

- [ ] ログイン画面がiPhone幅で崩れない
- [ ] マイページのカードが縦に自然に並ぶ
- [ ] プロフィール編集の入力欄が狭すぎない
- [ ] 公開/非公開トグルが押しやすい
- [ ] 保存ボタンが押しやすい
- [ ] 社員一覧カードが読みやすい
- [ ] 横スクロールが発生しない

## ビルド

- [ ] `npm run lint` が通る
- [ ] `npm run build` が通る
